## Creating Custom Dashboards

<Steps>
  <Step>
    ### Navigate to Dashboard Builder

    Go to **Analytics** â†’ **Dashboards** â†’ **Create Dashboard**
  </Step>

  <Step>
    ### Add Widgets

    Choose from available widget types:
    - **Metric Card**: Single key metric
    - **Line Chart**: Trends over time
    - **Bar Chart**: Comparisons
    - **Pie Chart**: Distribution
    - **Table**: Detailed data
    - **Heatmap**: Time-based patterns
  </Step>

  <Step>
    ### Configure Data Sources

    Select metrics and dimensions for each widget:

    ```yaml
    widget:
      type: line_chart
      metric: trace_count
      group_by: model
      time_range: 7d
      filters:
        - environment = "production"
    ```
  </Step>

  <Step>
    ### Save and Share

    Name your dashboard and set sharing permissions.
  </Step>
</Steps>

### Time Series Charts

Track trends over time:

```python
# Via API
chart_data = client.analytics.get_time_series(
    metrics=["trace_count", "token_count"],
    start_time=datetime.now() - timedelta(days=7),
    group_by="day"
)
```

### Comparison Charts

Compare dimensions side-by-side:

```yaml
widget:
  type: bar_chart
  metric: latency_p95
  group_by: model
  filters:
    - time_range = "24h"
```

### Distribution Charts

Show breakdowns:

```yaml
widget:
  type: pie_chart
  metric: cost
  group_by: model
```

### Data Tables

Detailed tabular data:

```yaml
widget:
  type: table
  columns:
    - trace_id
    - model
    - latency_ms
    - cost
    - quality_score
  sort_by: cost
  sort_order: desc
  limit: 100
```

### Heatmaps

Visualize patterns across time:

```yaml
widget:
  type: heatmap
  metric: trace_count
  x_axis: hour_of_day  # 0-23
  y_axis: day_of_week  # Mon-Sun
```

## Best Practices

### 1. Purpose-Driven Dashboards

Create focused dashboards for specific audiences:

```yaml
# Bad: One dashboard for everything
# Good: Separate dashboards per audience

engineering_dashboard:
  focus: performance, errors, latency

product_dashboard:
  focus: usage, features, engagement

executive_dashboard:
  focus: costs, ROI, trends
```

### 2. Use Consistent Time Ranges

Align time ranges across related widgets for accurate comparison:

```yaml
# All widgets should use the same time range
dashboard:
  default_time_range: "7d"
  widgets:
    - time_range: inherit  # Uses dashboard default
    - time_range: inherit
```

### 3. Add Context

Include comparison metrics for context:

```yaml
widget:
  type: metric
  metric: daily_cost
  comparison: "vs_same_day_last_week"  # Shows +15% context
```

### 4. Set Up Alerts from Dashboards

Link alerts to dashboard thresholds:

```python
# Create alert from dashboard metric
client.alerts.create_from_widget(
    dashboard_id="dash_123",
    widget_id="widget_456",
    threshold=100,
    operator=">"
)
```
